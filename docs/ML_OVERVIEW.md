## 一、核心思路（一句话 + 五个要点）

> **把"预估次日高低点"（监督、可验证）和"挂价决策"（规则→bandit→RL）拆开，用 1h 撮合 + 三基线把每一步的可信度钉死，能停则停、不为 RL 而 RL。**

### 1. 要复刻的人工动作
你现在：**看几个月趋势 → 预估次日最高/最低点 → 挂限价单（买偏低、卖偏高）+ 决定数量**。
目标函数 = **最大化达成交易净值（卖出额 − 买入额，税前毛额，移动平均成本口径）**。

### 2. 灵魂决策：拆成两层（不做端到端 RL 黑箱）

| 层 | 干什么 | 方法 | 为什么这么分 |
| --- | --- | --- | --- |
| **预测层** | 预测次日 [最低 L̂, 最高 Ĥ] 区间 | 监督学习 / 分位数回归 | 每天一个样本、样本相对足、可单独算误差、可解释 |
| **决策层** | 挂什么价、挂多少量（含不挂） | 规则 → bandit → RL | 这才是序贯/带约束的难点；把不确定性关在这一层 |

拆层的理由：预测层稳、可独立上线；决策层的风险被隔离。**一开始就端到端 RL → 无法 debug + 小样本必过拟合。**

### 3. 务实迭代，能停就停（RL 是研究性增量，不是起点）
```
S0 规则基线（永远保留做对照）
 → S1 预测器（监督，已可独立产出价值）
   → S2 bandit（LinUCB，比 RL 稳、样本效率高）
     → S3 RL（离线 RL / PPO，研究性，可能不达标）
```
**每阶段都有 go/no-go：打不过基线就停在上一阶段交付。**

### 4. 可信度的两个命门
- **撮合模拟器**：无逐笔数据 → 用 **1h K 线近似盘中限价撮合**（能判断盘中先触低还是先触高）。靠**真实 orders 回放校准**——实测吻合率 **88–90%**。
- **三基线门槛**：所有结果对照 ① 买入持有 ② 你的真实历史成交回放 ③ S0 规则。**绝不只报一条精挑曲线。**

### 5. 贯穿始终的纪律
- **防未来函数**：决策只用截至当日收盘可得信息；切分一律按时间 walk-forward，绝不随机打散。
- **敬畏小样本**：真实成交每标的仅 40 多笔 → 优先稳健可解释方法、强正则、多种子看方差。
- **单标的独立**：各自账户、各自本金，不共享现金池（第一版）。
- **工程解耦**：独立库 + 独立采集脚本 + 独立 conda env，**全程不碰 web/生产库**（写 web 是最后的 P6）。

## 二、训练用到的数据

**标的**：3 支美股（全 USD）—— **NVDA 英伟达 / TSLA 特斯拉 / PDD 拼多多**。

**独立库** `data/ml/mystock_ml.db`（与 web 生产库 `mystock.db` 解耦；已 gitignore）：

| 数据表 | 行数 | 覆盖 | 用途 |
| --- | --- | --- | --- |
| `ml_quotes_1d` 日线 | 3765（各 1255） | **2021-06 ~ 2026-06，5 年** | 预测层(P2)特征 + 标签 |
| `ml_quotes_1h` 1 小时线 | 15237（各 5079） | **2023-07 ~ 2026-06，约 2 年** | 撮合模拟器(P1)盘中路径 |
| `ml_deals` 成交快照 | 129（NVDA45/TSLA43/PDD41） | 2025-01 ~ 2026-06 | P1 校准 + P3 human 回放 |
| `ml_orders` 委托快照 | 205（72/74/59） | 同上 | P1 撮合吻合率校准 |
| `ml_positions` 持仓快照 | 6 | 2 个快照日 | 当前状态（时序基本没有） |

> 行情来自 yfinance（`auto_adjust=False`，留 close + adj_close，指标用 adj_close）；
> 交易事实从生产库 `mystock.db` **只读快照**拷贝，保证训练可复现、不在生产库跑分析。

### 各阶段实际"吃"了多少
- **P1 撮合校准**：真实 `orders` 72/74/59 单回放，全部落在 1h 覆盖窗内（无 bar=0）。
- **P2 预测器**：每标的 1255 日线 → 去特征窗(~20)+次日标签 → **≈1234 个可决策样本/标的**，walk-forward 4 折。
- **P3 回测**：`train_frac=0.6` → 训练 ≈740 日、**测试 ≈494 日**，三支独立账户各初始 2 万。

### 三个必须记住的数据约束（影响结论可信度）
1. **成交样本很小**（每标的 40 多笔、卖出十几到二十几笔）→ 小样本过拟合、bandit 难稳超基线的根源。
2. **1h ≠ 逐笔**：一根 1h 内 high/low 先后仍不可知（校准吻合 88–90%，残余失真已标注）。
3. **行情区间单一**：2024–2026 一段 regime，NVDA/TSLA 强相关、PDD 较独立；跨 regime 未必稳。
4. **human 回放非完全同口径**：真实成交只覆盖测试区间内 2025-01 起的一段；人类只在部分日子交易，策略每天可决策 —— 解读"bandit vs human"时需记得。

## 三、当前进度与已验证结论

| 阶段 | 内容 | 状态 | 关键数字 |
| --- | --- | --- | --- |
| **P0** | 数据底座（独立库 + 采集脚本 + H20 env） | ✅ | 见上表；H20 prefix env 已调通 |
| **P1** | 1h 撮合模拟器 + orders 校准 | ✅ | **校准吻合率 NVDA 90.3% / TSLA 87.8% / PDD 89.8%** |
| **P2** | 次日高低分位预测器 | ✅ | **区间命中率 76–85%**（预测区间包住次日实际高低） |
| **P3** | 规则基线 + bandit 回测 | ✅ | 见下表 —— **bandit 尚未稳定超基线** |
| **P4** | RL（研究性） | ⏸ 待定 | 环境已铺好，可选 |
| **P5** | 每日 HTML 报告 | ◻ 未开始 | 归档 `data/ml/reports/<date>/` |
| **P6** | 写入 web | ◻ 最后一步 | 达标稳定后才做 |

### P3 回测结果（494 测试日，期末账户净值，初始 2 万）—— 诚实记录
| 标的 | rule(S0) | bandit(S2) | human回放 | buy&hold | 行情 |
| --- | --- | --- | --- | --- | --- |
| NVDA | 25040 | 28858 | 24206 | **33638** | 单边强涨 |
| TSLA | 28244 | 18606 | 25420 | **38750** | 单边强涨 |
| PDD  | 19767 | 18974 | 20109 | 11840 | 震荡下行 |
（本机 sklearn 后端；H20 lightgbm 后端数字略异、结论相同。）

**诚实结论**：① 强单边涨段（NVDA/TSLA）buy&hold 难被击败；② 下行/震荡段（PDD）所有交易策略均赢 buy&hold，说明择时在非单边行情有价值；③ **bandit 尚未稳定超 S0 规则** → 当前 go/no-go **不达标**，按门槛先停在 S0 + 预测层交付，RL 作研究性尝试不强推。

## 四、代码与运行

| 文件 | 职责 |
| --- | --- |
| `mystock/ml/fetch.py` + `scripts/ml_fetch.sh` | 数据采集（独立于 update.sh） |
| `mystock/ml/data.py` | ML 库只读对齐 |
| `mystock/ml/simulator.py` | 1h 撮合 + 单标的账户 |
| `mystock/ml/calibrate.py` | 真实 orders 回放校准 |
| `mystock/ml/features.py` | 特征工程（防未来函数、用 adj_close） |
| `mystock/ml/predictor.py` | 次日高低分位回归（LightGBM 优先、sklearn 回退） |
| `mystock/ml/policy.py` | S0 规则 + S2 LinUCB bandit |
| `mystock/ml/backtest.py` | 逐日回放 + 三基线对照 |
| `scripts/ml_setup_h20.sh` | H20 持久 prefix conda env（libgomp + cu129 torch） |
| `scripts/ml_sync_h20.sh` / `ml_vllm.sh` | 同步数据 / vllm 让位 |

```bash
# 本机（mk 环境）
bash scripts/ml_fetch.sh                  # 采集 → data/ml/mystock_ml.db
python -m mystock.ml.calibrate            # P1 校准
python -m mystock.ml.predictor            # P2 预测器
python -m mystock.ml.backtest             # P3 回测 vs 三基线
python -m pytest tests/test_ml_*.py -q    # 单测

# H20（持久 prefix env）
conda activate /home/share/user/chenkai/mystock-ml/env
```

> 测试：撮合/账户/策略均为纯函数，已单测覆盖（`tests/test_ml_simulator.py`、`tests/test_ml_policy.py`）。
