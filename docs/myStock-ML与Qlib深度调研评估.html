0. TL;DR 裁决

不要把 myStock ML 整体迁到 Qlib，也不要用 Qlib 的组合选股管线替换现有两层架构。 问题定义不同：myStock 是「单标的次日高低区间 → 限价挂单」，Qlib 默认是「截面 Alpha → TopK 组合」。

正确用法：把 Qlib 当只读的方法库 / 实验沙箱，分组件借鉴—— 优先特征与预测评估，其次滚动适应与实验纪律， 最后才考虑订单执行 RL 思想。现有预测层、1h 撮合、CQR、regime、三基线纪律应保留并强化。

立刻可做（P0）

Alpha 特征扩容 + IC/ICIR 评估；预测层多模型对照（LGBM 已有，可加线性/MLP 基线）；多种子方差入报告。

中期可做（P1）

滚动重训 / 概念漂移适应；可选把日线 dump 成 qlib bin 做离线实验；OPE 闸门。

明确不做

端到端 qrun 替换决策层；TopkDropout 组合策略；跨股共享资金池；把 pyqlib 塞进 cron 主链路（除非独立 opt-in）。

2.3 工程纪律（强项，必须保留）

  • 两层拆分：预测可独立验证，决策风险隔离 —— 已被实验验证正确。
  • 防未来函数：特征只用 T 收盘；HMM 用前向算法而非 forward-backward。
  • 三基线门槛：B&H / 人类回放 / S0 规则永远对照。
  • 工程解耦：ML 库与 web 生产库物理分离；采集不进 update.sh。
  • 脏数据三层防护：采集 / 回测 / 报告防 NaN 污染。
  • 能停则停：RL 负结果如实记录，不粉饰。

4.2 根因清单（按影响排序）

  1. 决策问题比预测难一个量级：单边牛市 B&H 极难击败；限价策略本质是「少交易、等波动」——与「天天可决策」的 bandit 设定存在张力。
  2. 特征偏薄：16 维多是短中期动量/波动，缺少截面相对强度、更长记忆、形态/量价交互；限制了预测与上下文 bandit 的上界。
  3. 评估指标偏结果、偏单次：期末净值是必要的，但缺少 IC 序列、分 regime 拆账、多种子 std、配置敏感性表。
  4. 小样本硬天花板：真实成交 40–110 笔/股；离线 RL 已实证失败；继续堆深度模型期望为负。
  5. 测试窗与行情 regime 绑定：2024–2026 强趋势主导，策略排序会随窗移动。

5. Qlib 能力地图（与 myStock 相关切片）

Qlib 是微软开源的 AI 量化平台：数据层 → 学习框架 → 工作流 → 分析。完整中文介绍见 Qlib中文详细介绍.html。此处只列可对齐 myStock 的切片：

数据与特征

.bin 存储、表达式引擎、Alpha158/360、Processor 标准化、PIT 防未来财报。

预测与模型 Zoo

LGBM/XGB/LSTM/Transformer/HIST… + CSI300 上 IC/IR 对照表；qrun 一键 workflow。

动态适应

Rolling retrain、DDG-DA 元学习；针对非平稳市场。

决策与 RL

TopK 组合、Nested 日度+日内、订单执行 PPO/OPDS；分析层 IC/回撤图。

Qlib 最强项对 myStock 的含义 Qlib 最成熟的是截面 Alpha 研究工厂（特征 + 多模型 + 标准化评估）。 myStock 最缺的正是「特征上界」与「评估标准化」——这是最大交集。 Qlib 的组合回测与 myStock 的限价决策不是同一产品，交集反而小。

8. 借鉴策略总纲：三种集成模式

模式 A — 思想移植（推荐默认）

不引入 pyqlib 依赖。把 Alpha 公式、IC 指标、滚动重训伪代码用现有 pandas/lightgbm 重写进 mystock/ml/。 优点：零耦合、cron 不破、测试可控。缺点：要手写表达式。

模式 B — 离线实验沙箱（推荐中期）

把 ml_quotes_1d dump 成 Qlib .bin（仅研究机），用 qrun / notebook 做： 特征筛选、模型对照、IC 曲线。结论再回写进 myStock 特征列表与超参。 Qlib 永不进入 ml.sh all 主链路。

模式 C — 运行时硬依赖（不推荐现阶段）

import qlib 进预测器/回测。问题：依赖重、版本钉死难、与「小而稳的 CPU 例行」冲突，且问题形式仍要大量定制 adapter。

本报告推荐 A 为主、B 为辅、C 不做。 Qlib 仓库保持只读；myStock 继续拥有数据与决策主权。

12. 验收 KPI（「优化有效」的可操作定义）

避免「换了个酷模型但不可复现」。建议把成功定义成下面可同时满足的集合：

预测层 KPI

  • CQR 测试窗覆盖率 ≥ 目标覆盖率（现 70%）± 合理松弛（≥65%）。
  • 平均区间宽度不比当前配置恶化超过 +15%（相对 close）。
  • 新增：|IC| 对 high 或 low ret 在 ≥4/6 股显著异于 0（时间序列 t 检验或简单符号稳定）。

决策层 KPI

  • 条件达标：在「非强单边」regime 子集上，Bandit 期末 ≥ max(S0, B&H) 的标的数 ↑。
  • 不伤牛市：强趋势段 Bandit 相对 B&H 的落后幅度收窄（门控生效）。
  • 多种子：5 seed 下「超越 B&H」结论一致，或明确标为不稳定。
  • 永远对照 S0 / human / B&H；不允许只报最优曲线。

工程 KPI

  • pytest 全绿；新增因子/指标有纯函数单测。
  • ml.sh all 本机 CPU 可跑完；Qlib 不进入默认路径。
  • 报告自解释：配置行写清 reward / CQR / regime / 特征版本。
