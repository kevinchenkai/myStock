## 0. TL;DR（结论先行）

- **目标可形式化**：把"人类看几个月趋势 → 预估次日最高/最低 → 挂限价单买/卖 + 数量"这个动作，建成一个**每日决策**的序贯问题；优化目标=**最大化每个买卖回合的达成净值（卖出额 − 买入额，税前毛额）**。
- **小样本仍是头号风险，但已被"可抓更多数据"显著缓解**。实测 yfinance：日线可取 **5 年**（1255 行/股 vs 现库 ~366），1 小时线可取 **2 年**（覆盖全部成交窗口）。→ 预测层样本量 ×3.4；撮合模拟器可从"日线近似"升级为 **真实 1h 盘中路径**，保真度质变（见 §2.1、§4）。
- **务实迭代路线（用户已认可）**：
  1. **S1 预测层**：次日高/低区间预测（监督/分位数回归），直接复刻人类在做的事，可独立验证、独立产出；
  2. **S2 决策层（bandit）**：在 1h 撮合模拟器上学"挂价档 + 数量档"以最大化回合净值；
  3. **S3 RL（研究性增量）**：离线 RL（IQL/CQL，用真实 deals 当数据集）或模拟器 PPO；**跑不赢基线就如实记录、不上线**。
- **基线门槛**：全程对照 ① 买入持有、② 人类历史成交回放、③ S0 规则。**打不过就停在上一阶段交付。**
- **H20 算力富余**：模型小数据小，单卡分钟级一轮。2 卡用于**并行铺种子/超参/标的**对抗小样本高方差，而非堆大模型。

> 一句话：**可行性较初版上调**——靠"扩数据 + 1h 撮合"把两大风险（样本量、撮合失真）都压下来了；仍按"敬畏小样本、严守基线门槛、能停则停"的方式做。**工程上完全与 web 解耦**（独立库、独立采集脚本、独立 conda env），不影响现有系统。

## 1. 问题定义

### 1.1 人类当前在做什么（要复刻的对象）
每个交易日收盘后 / 开盘前：
1. 看该标的过去数月走势；
2. 预估**次日最高点 H_hat、最低点 L_hat**；
3. 据此挂**限价单**：买单挂在偏低位、卖单挂在偏高位（数据已证实这一行为，见 §2.3）；
4. 决定**数量**。

### 1.2 拆成两层（关键设计决策）
| 层 | 子问题 | 方法候选 | 能否独立验证 |
| --- | --- | --- | --- |
| **A. 预测层** | 次日 [L_hat, H_hat] 区间（及不确定性） | 分位数回归 / 概率预测（监督学习） | ✅ 直接对真实次日 high/low 算误差 |
| **B. 决策层** | 给定预测与持仓，挂买/卖价 + 数量（含不挂） | 规则 → contextual bandit → RL | 需撮合模拟器回测 |

**为什么拆**：预测层是监督问题、样本相对充足（每标的每天一个样本）、可单独评估、可解释；决策层才是序贯/带约束的部分。把不确定的 RL 局限在 B 层，A 层用稳健监督模型，能极大降低过拟合面与调试难度。**如果一开始就端到端 RL，几乎无法 debug**。

## 1.5 范围与目标（已锁定）

- **标的**：3 支**美股**——**NVDA（45 成交）/ TSLA（43）/ PDD 拼多多（41）**。全 USD 计价，免跨币种归一。PDD 现库 `cost_price=28.98` 为正（NVDA/TSLA 成本含 ≤0 会计产物），数据质量更干净。
- **目标函数**：**最大化达成交易净值 = Σ(卖出成交额) − Σ(买入成交额)**（税前毛额，无费用字段）。等价于按**移动平均成本**口径的已实现盈亏最大化（沿用 `pnl.py`）。
- **迭代策略**：S1→S2→S3 逐步推进，**每阶段设 go/no-go**，能停则停，不强推 RL。
- **允许扩数据**：可调 yfinance 抓更长历史 / 更细粒度（见 §2.1），但仍**只读现库 + 新增独立采集脚本**，不破坏"抓取与展示分离"的项目边界。

## 2. 数据可行性核查（基于真实库只读 + yfinance 实测）

### 2.1 现有量级 vs 可扩量级（**本版关键更新**）

**现库快照（2026-06-18/22）**：
- `daily_quotes`：13491 行，每标的 ~358（HK）/366（US）交易日，2025-01-02 起。
- `deals`：873 条真实成交，34 只标的；3 支目标 US 的成交窗口 2025-01 ~ 2026-06。
- `positions`：仅 2 个快照日——**持仓时序基本没有**，只能当"当前状态"。

**实测可抓取（yfinance，针对 NVDA/TSLA）**：
| 粒度 | 可取窗口（硬限制） | 实测行数 | 用途 |
| --- | --- | --- | --- |
| **日线 1d** | **5 年** | 1255 行/股（2021-06 起） | 预测层训练样本 **×3.4** |
| 1m | 仅最近 7 天 | ~2500 | 太短，不用于训练 |
| 5m / 15m | 最近 60 天 | ~4600 / ~1500 | 近期校准/回测补充 |
| **1h（60m）** | **2 年**（约 2023-07 起） | ~5000 行/股 | **撮合模拟器的盘中路径**，覆盖全部成交窗口 |

→ **两个直接收益**：
1. **预测层样本量翻 3 倍以上**（5 年日线），缓解过拟合。
2. **撮合从"日线近似"升级为"1h 盘中真实路径"**：能判断盘中**先触 low 还是先触 high**、限价是否真被触达——撮合保真度质变（见 §4）。1h 覆盖 2023-07 至今，完整包住 deals 窗口（2025-01~）。

### 2.2 仍需在设计层面应对的限制
1. **盘中仍非逐笔**：1h K 线内部顺序未知（一根 1h 内 high/low 谁先到仍不可知），但比纯日线已大幅改善。逐笔/分钟级历史不可长期回溯（1m 仅 7 天）→ 接受 1h 近似并标注局限。
2. **样本仍偏小 + 标的强相关**：3 支里 NVDA/TSLA 高相关，PDD（中概）相对独立。按时间切分、警惕泄漏；多种子评估方差。
3. **窗口外历史缺失 + 成本异常**：`deals` 从 2025-01 起，更早买入缺失；`positions.cost_price` 有 ≤0 会计产物（如 AAPL −1233、BA −713、目标 3 支中 NVDA cost 亦为负）。→ 沿用 `pnl.py`：`cost_price≤0` 记 uncovered、不产奖励；回合净值口径以 deals 内配对为准。
4. **复权一致性**：扩抓的 5 年日线 `auto_adjust=False`，保留 close + adj_close，与现库口径一致；技术指标用 adj_close。

### 2.3 训练数据单独建库（与 web 完全解耦）

**原则**：web 层只读生产库 `data/mystock.db`（DATA.md §0）；ML 训练**绝不写它、也不被它的 schema 约束**。

- **新库**：`data/ml/mystock_ml.db`（独立 SQLite，gitignore 同 `data/`）。
- **数据来源关系**：
  - 行情（扩抓的 5 年日线 + 2 年 1h）→ **写 ML 库**（生产库放不下这些粒度，schema 也不同）。
  - 交易事实（`deals` / `orders` / `positions`）→ **从 `mystock.db` 只读拷贝一份快照**进 ML 库（或建只读 ATTACH），保证训练可复现、且不在生产库上跑分析查询。
- **ML 库 schema（草案，独立于 schema.sql）**：
  - `ml_quotes_1d(symbol, date, open, high, low, close, adj_close, volume, dividends, splits)`
  - `ml_quotes_1h(symbol, ts_utc, open, high, low, close, volume)` — ts 存 UTC，附 `ts_et` 便于核对
  - `ml_deals` / `ml_orders` / `ml_positions`：生产库对应表的快照拷贝（含 `snapshot_taken_at`）
  - `ml_features(symbol, date, ...)`：特征工程产物（落库便于复现与多次训练复用）
  - `ml_runs(run_id, stage, config_json, metrics_json, created_at)`：训练运行登记（种子/超参/指标）
- **好处**：① web 永远不受训练影响；② 训练数据冻结可复现；③ 粗细粒度行情分表互不干扰；④ 删库重训不碰生产数据。

### 2.4 扩充数据采集脚本独立实现（不并入 update.sh）

**原则**：`scripts/update.sh` 维持原职责（富途 + yfinance 日线 → 生产库），**完全不动**。ML 采集是另一条独立管线。

- **新增**：`scripts/ml_fetch.sh` + `mystock/ml/fetch.py`（或 `mystock/ml/` 子包）。
- **职责**：
  1. 拉 NVDA/TSLA/PDD 的 **5 年日线**（`period=5y, interval=1d, auto_adjust=False`）→ `ml_quotes_1d`。
  2. 拉 **2 年 1h**（`interval=60m`，分段绕过窗口限制、合并去重）→ `ml_quotes_1h`。
  3. 从 `data/mystock.db` **只读快照** deals/orders/positions → `ml_deals/...`。
  4. 全部 UPSERT、幂等；写自己的 `ml_sync_log`，不碰生产 `sync_log`。
- **复用**：可 import `mystock/code_map.py` 等纯函数，但**不复用 update_load 的写库路径**（避免耦合）。
- **配置**：标的清单、抓取窗口、库路径写在 `mystock/ml/config`（或复用 config.yaml 增独立段），不混入现有 pipeline 配置。
- **运行**：`bash scripts/ml_fetch.sh`（本机抓数 → 生成 `mystock_ml.db` → `scripts/ml_sync_h20.sh` 同步到私人 H20）。

### 2.5 人类行为可学（已验证，支持立项）
抽样腾讯成交相对当日 OHLC 的位置（pos = (price−low)/(high−low)）：
- **BUY** 落在 0.13–0.43（偏低买）；**SELL** 落在 0.75–0.93（偏高卖）。

## 3. 状态 / 动作 / 奖励设计（决策层 B）

### 3.1 State（每标的每日，收盘后构造，预测次日）
全部用**可在 T 日收盘后获得**的信息，严防未来函数：
- 行情因子（用 `adj_close`，5 年日线）：近 N 日收益、波动率、ATR、均线偏离、量比、当日 OHLC 相对位置、近 5/20 日 high/low。
- 区间预测（S1 层输出）：L_hat、H_hat、预测宽度/不确定性。
- 持仓上下文：当前 qty、可卖 qty、移动平均成本、浮盈浮亏、距成本距离、是否 uncovered（cost≤0）。
- 标的画像（低频，3 支 US 均 USD）：Beta、市值分层、PE/PB 分位（缺失值策略见 DATA.md §5）。
- （第一版仅 US，无需币种归一。）

### 3.2 Action
离散+受限，先简单：
- 方向：{不动, 买, 卖}；
- 挂价：相对 [L_hat,H_hat] 的**分位档**（如买挂 L_hat、L_hat+0.25w…；卖挂 H_hat−…）——动作空间小、可解释；
- 数量：相对持仓/可用资金的**离散档**（0/25%/50%/100% 等）。
- 硬约束：卖不超过 `can_sell_qty`；买受可用现金约束；限价单口径与项目一致。

### 3.3 Reward（对齐"最大化达成交易净值"）
- **主奖励**：每当一个买卖回合（买入后卖出）闭合，奖励 = **该回合净值 = 卖出额 − 买入额**（税前毛额，移动平均成本口径，沿用 `pnl.py`）。未闭合的持仓按期末市值折算浮动部分，避免"只买不卖"刷状态。
- **撮合判定**：由 §4 的 **1h 盘中路径模拟器**给出"是否成交、成交价"，而非日线近似。
- **风险/行为惩罚（可选项，调参开关）**：对过度换手、满仓单边、长期不平仓加小惩罚，防止学出脆弱策略。
- **奖励的意义看相对值**：最终以"策略累计净值曲线 − 三基线"判优劣，绝对收益不单独采信。

### 3.4 资金约束：**单标的独立**（已锁定）
- 每支标的一套独立账户：**各自给定初始名义本金**，自己的现金与持仓，**互不借调资金**。
- 三支并行回测但**不共享现金池**——简化第一版、去掉组合级资金调度的复杂度。
- 训练/评估各标的独立成 episode；汇总报告时再把三条净值曲线分别 + 合并展示。
- （组合级现金共享、跨标的再平衡留作后续增量，非第一版范围。）

## 4. 训练环境（模拟器）—— 本方案的工程核心

### 4.1 为什么必须自建 env
公开 gym 交易环境不适配我们"**1h 盘中限价撮合 + 富途持仓/移动平均成本口径 + 回合净值奖励**"。需要一个**轻量、确定、可单测的 `gymnasium.Env`**。

### 4.2 撮合规则：**1h 盘中路径**（替代日线近似）
对 T+1 日，回放该日**逐根 1h K 线**（yfinance 可取 2 年 1h，覆盖成交窗口）：
- 把次日 1h K 线按时间顺序遍历；对每根 bar 用其 [low, high] 判定限价是否被触达：
  - **限价买** P_b：第一根满足 `low(bar) ≤ P_b` 的 bar 成交，成交价 = `min(P_b, open(bar))`。
  - **限价卖** P_s：第一根满足 `high(bar) ≥ P_s` 的 bar 成交，成交价 = `max(P_s, open(bar))`。
- 相比纯日线，**能正确处理"盘中先到 low 还是先到 high"的顺序**（只在同一根 1h 内仍有残余不确定）。
- 一日一单（先简化），当日未触达则失效（对应人类当日有效单）。
- 剩余近似误差：1h bar 内部顺序、部分成交、滑点、停牌——如实标注为局限。
- **历史早于 1h 覆盖窗（2023-07 前）** 的样本：S1 预测层仍可用 5 年日线训练；S2/S3 撮合回测**只在 1h 覆盖区间内做**。

### 4.3 撮合校准（结论可信度的命门）
开训前先做一次校准：拿**真实 `orders` 表的挂价 + `order_status`**，用上述 1h 规则回放，看"模拟成交与否"与"真实 FILLED/CANCELLED"的吻合率。吻合率达标后再用于训练，并把吻合率写进报告。

### 4.4 数据切分（防过拟合第一道闸）
- **按时间** walk-forward 切：如 2024-06~2025-09 训练 / 2025-10~2026-01 验证 / 2026-02~06 测试（滚动多折）。
- 绝不随机打散按行切（时序泄漏）。
- 3 支标的各自独立账户（§3.4）、合并训练共享模型权重，但**测试时间段对三者统一**；NVDA/TSLA 高相关，评估时单独看 PDD 以免被两只大盘主导。

## 5. 口径与清洗（采用 DATA.md，按"3 支 US"裁剪）
1. 收益率/技术指标一律用 `adj_close`，不用 `close`（含新抓的 5 年日线，`auto_adjust=False`）。
2. 跨表 JOIN 用 `futu_code`（`US.NVDA/US.TSLA/US.PDD` ↔ yf `NVDA/TSLA/PDD`）。
3. 第一版仅 US、全 USD，**免跨币种归一**；`deals` 无 currency，用 market=US→USD。
4. `positions.cost_price ≤ 0` → uncovered，不入成本、不产奖励（NVDA 现库 cost 即为负）。
5. 新抓数据按 date 与现库对齐；1h 与 1d 两套表分开存，回测按粒度取用。
6. 时区：美股成交/行情统一按 America/New_York 处理（库内无时区标记，单市场内排序无歧义）。
7. 前端展示沿用**红涨绿跌**；本方案产出图表同此。
8. **隐私**：H20 为私人机器、数据安全，可直接 rsync 真实交易数据；但**产物绝不进公开 git 仓库**（`data/` 已 gitignore，DATA.md §8.3.9）。

## 6. 算法选型（从稳到激进，建议按序推进，能停就停）

| 阶段 | 决策层方法 | 何时够用 | 风险 |
| --- | --- | --- | --- |
| **S0 基线** | 规则：买挂 L_hat、卖挂 H_hat、固定量 | 永远保留，做对照 | 无 |
| **S1 预测器** | 分位数回归 / LightGBM / 小型时序网络 预测次日 high/low（5 年日线） | 独立产出即有用 | 小样本，需正则+walk-forward |
| **S2 Bandit** | Contextual bandit 选"挂价档+数量档"以最大化回合净值（单步） | 决策可独立于序贯依赖时 | 比 RL 稳，样本效率高 |
| **S3 RL** | 离线 RL（IQL/CQL，用真实 deals 当离线数据）或 PPO on 1h 模拟器 | S1/S2 跑通且撮合校准后 | 过拟合、模拟器偏差、信用分配 |

**务实定位（用户已认可）**：交付价值压在 **S1+S2**，**S3（RL）作研究性增量**。RL 在数百条成交上做 model-free PPO 方差极大；若做 RL，优先**离线 RL**——直接拿真实成交/挂单当数据集，避免在线探索在小样本乱走。**任一阶段打不过基线，就停在上阶段交付并如实记录。**

## 7. 评估（决定上不上线的唯一依据）
- **主指标（对齐目标）**：测试区间内**累计达成交易净值（Σ卖出−Σ买入，含期末持仓折算）**，对照
  1. 买入持有，2. 人类历史成交（同一 1h 撮合规则回放真实 deals），3. S0 规则。
- **预测层 S1**：次日 high/low 的 MAE / 分位数 pinball loss / 区间命中率，按时间外推集。
- **稳健性**：多随机种子（≥5）方差、walk-forward 多折一致性、3 支分别看（防 NVDA/TSLA 主导）。
- **门槛**：**净值跑不过三基线 / 方差巨大 → 判不可上线，如实报告。绝不只报一条精挑曲线。**

  - 每日一份**不可变快照**：历史报告不回写，方便横向对比模型随时间的表现。
