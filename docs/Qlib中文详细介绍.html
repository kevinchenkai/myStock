1. 项目是什么

Qlib 是微软亚洲研究院开源的面向 AI 的量化投资平台。 它把「从想法探索到生产落地」整条量化投研链路封装成可组合的模块：数据处理、特征工程、 模型训练、策略生成、回测执行、结果分析、在线滚动更新与订单执行，一站式覆盖。

官方定位（摘要）：

  • 用 AI 技术挖掘量化投资的潜力，赋能研究，创造价值；
  • 支持多种机器学习范式：监督学习、市场动态建模、强化学习；
  • 覆盖完整 ML Pipeline：数据处理 → 模型训练 → 回测评估；
  • 覆盖完整量化链路：Alpha 挖掘 → 风险建模 → 组合优化 → 订单执行。

学术论文： Qlib: An AI-oriented Quantitative Investment Platform（arXiv:2009.11189）。

安装包名是 pyqlib（不是 qlib），导入时仍写 import qlib。

2. 为什么用 Qlib

研究友好

用 YAML 配置 + qrun 就能跑通「特征 → 训练 → 回测 → 分析」全流程；也可用代码自由拼装。

高性能数据层

自研 .bin 二进制存储 + 表达式引擎，适合海量股票截面/时序计算；支持日频与 1 分钟高频。

模型 Zoo 丰富

内置 LightGBM / XGBoost / LSTM / Transformer / HIST 等 20+ 量化模型实现与可复现 benchmark。

生产链路

支持实验记录（Recorder/MLflow）、在线 serving、模型自动滚动更新、RL 订单执行等生产向能力。

适合场景 个人/团队量化研究、Alpha 因子挖掘与对比、组合回测、高频与订单执行 RL、 需要可复现论文实验的学术/工程团队。
不适合 / 需注意 官方公开数据来自 Yahoo 等公开源，质量有限；严肃交易请换自有高质量数据。 本平台是研究框架，不是券商交易终端，也不提供投资建议。

3. 量化研究核心难题与 Qlib 的解法

金融场景与普通 ML 任务差异很大。Qlib 针对几类关键挑战给出了可运行方案：

挑战问题本质Qlib 中的解法
预测 / Alpha 挖掘 金融数据量大、异构、非线性，信号难挖 Quant Model Zoo（树模型 / 深度学习 / 图模型等）+ Alpha158/360 特征集
市场非平稳 训练分布与未来测试分布漂移（concept drift） Rolling 重训、DDG-DA 等动态适应方案（examples/benchmarks_dynamic）
连续决策 交易是序列决策，不是单次预测 RL 框架 + 订单执行（TWAP / PPO / OPDS 等）
多层决策嵌套 日度组合管理 + 日内订单执行需协同 Nested Decision / NestedExecutor 框架
可复现与追踪 实验难复现、难对比 Workflow + Recorder（MLflow）统一记录模型、预测、回测产物

4. 整体架构

Qlib 采用松耦合模块设计：每一层、每个组件都可以单独使用，也可以拼成完整投研流水线。

┌─────────────────────────────────────────────────────────────┐ │ Interface 层（界面 / 分析） │ │ Analyser · 图形报告 · CLI(qrun) · Notebook │ ├─────────────────────────────────────────────────────────────┤ │ Workflow 层（投研全流程） │ │ Information Extractor → Forecast Model → Decision Generator │ │ → Execution Env（可嵌套 Strategy / Executor） │ ├─────────────────────────────────────────────────────────────┤ │ Learning Framework 层（学习范式） │ │ 监督学习（Forecast Model） · 强化学习（Trading Agent） │ ├─────────────────────────────────────────────────────────────┤ │ Infrastructure 层（基础设施） │ │ DataServer（.bin / 表达式） · Trainer · Cache │ └─────────────────────────────────────────────────────────────┘

高层抽象图见仓库：docs/_static/img/framework-abstract.jpg； 更细的框架图：docs/_static/img/framework.svg。

5. 四层结构详解

5.1 Infrastructure（基础设施）

  • DataServer：高性能读写原始行情/因子，表达式引擎支持 $close、Ref($close, 1)、Mean($close, 3) 等公式化 Alpha。
  • Trainer：灵活控制训练过程，便于算法自定义训练循环。
  • 底层滚动/扩展窗口算子用 Cython 加速（qlib/data/_libs/*.pyx）。

5.2 Learning Framework（学习框架）

  • 监督学习：训练 Forecast Model，输出 Alpha / 风险等信号。
  • 强化学习：训练 Trading Agent / Policy，端到端生成交易决策。
  • 可与 Workflow 共享 Information Extractor、Execution Env；支持 NestedExecutor 做多层策略联合优化。

5.3 Workflow（工作流）

  1. Information Extractor：抽特征 / 标签；
  2. Forecast Model：产生预测信号；
  3. Decision Generator：根据信号生成组合/订单（或 RL Policy 直接决策）；
  4. Execution Env：模拟/对接市场执行；
  5. 多层 Strategy + Executor 可嵌套（例如：日度组合策略里嵌套日内订单执行策略）。

5.4 Interface（接口）

  • qrun 配置驱动自动执行；
  • Python API 代码驱动自定义流程；
  • Analyser 输出 IC、累计收益、回撤、多空组合等分析报告。

10. Workflow 与 qrun

qrun 根据 YAML 自动执行一次完整 execution：

  1. Data：加载 → 处理 → 按时间切分 train/valid/test
  2. Model：训练、推理、保存/加载
  3. Evaluation：信号分析 + 组合回测

全程产物由 Recorder（常对接 MLflow）记录，便于对比实验。

10.1 典型配置结构（LightGBM + Alpha158）

qlib_init:
    provider_uri: "~/.qlib/qlib_data/cn_data"
    region: cn
market: &market csi300
benchmark: &benchmark SH000300

data_handler_config: &data_handler_config
    start_time: 2008-01-01
    end_time: 2020-08-01
    fit_start_time: 2008-01-01
    fit_end_time: 2014-12-31
    instruments: *market

task:
    model:
        class: LGBModel
        module_path: qlib.contrib.model.gbdt
        kwargs: { loss: mse, learning_rate: 0.2, ... }
    dataset:
        class: DatasetH
        module_path: qlib.data.dataset
        kwargs:
            handler:
                class: Alpha158
                module_path: qlib.contrib.data.handler
                kwargs: *data_handler_config
            segments:
                train: [2008-01-01, 2014-12-31]
                valid: [2015-01-01, 2016-12-31]
                test:  [2017-01-01, 2020-08-01]
    record:
        - class: SignalRecord      # 保存预测
        - class: SigAnaRecord      # IC 等信号分析
        - class: PortAnaRecord     # 组合回测分析

10.2 调试模式

python -m pdb qlib/cli/run.py examples/benchmarks/LightGBM/workflow_config_lightgbm_Alpha158.yaml

10.3 自定义模型接入

实现 Qlib 模型接口（通常继承 qlib.model.base.Model，实现 fit / predict）， 在 YAML 里把 class / module_path 指到你的模块即可。详见文档 Custom Model Integration。

11. 数据层深入

典型数据流水线：

原始 CSV / 采集源 │ dump_bin / collector ▼ .bin 存储（OHLCV 等基础字段） │ 表达式引擎（Data Loader） ▼ 公式化特征（如 Ref($close,60)/$close） │ Processor（Data Handler） ▼ 清洗 / 标准化 / 去极值 │ Dataset ▼ 模型可用的 train/valid/test 张量或 DataFrame

11.1 核心抽象

组件作用
D（Data API）日历、股票池、特征查询的统一入口
Expression Engine用类公式 DSL 写 Alpha；算子见 qlib/data/ops.py
Data Loader按表达式从存储取数
Data Handler组织特征/标签，挂 Processor 做复杂预处理
Processor归一化、缺失值、截面 rank 等（表达式不好表达的逻辑）
Dataset按模型需要切分、采样、构造输入
Cache表达式与处理结果缓存，加速反复实验
PITPoint-in-Time 数据库，避免财报等未来函数

11.2 离线模式 vs 在线模式

  • Offline（默认）：本地读 .bin，适合单机研究。
  • Online：可部署数据服务，多客户端共享；适合团队/生产。

13. 策略与回测

13.1 常用策略：TopkDropoutStrategy

每日根据预测分数选 Top-K 股票持有；同时强制换掉得分最低的 N 只（n_drop），控制换手。

strategy:
    class: TopkDropoutStrategy
    module_path: qlib.contrib.strategy
    kwargs:
        signal: <PRED>
        topk: 50
        n_drop: 5

13.2 回测关键参数

参数含义
account初始资金
benchmark基准（如 SH000300）
limit_threshold涨跌停阈值（A 股常 0.095）
deal_price成交价字段（如 close）
open_cost / close_cost买卖费率
min_cost最低交易成本

14. 强化学习

Qlib 的 RL 用于建模连续投资决策，尤其是订单执行（Order Execution）场景： 在给定母单下，学习如何拆单、择时，以最小化冲击成本 / 跟踪误差等。

14.1 场景与基线

方法说明
TWAP时间加权平均基准
PPOIJCAI 2020 端到端最优交易执行框架
OPDSAAAI 2021 Oracle Policy Distillation

示例目录：examples/rl_order_execution/；文档： RL Component。

14.2 框架特点

  • 用 Workflow 的 Execution Env 构造 RL 环境；
  • 支持 NestedExecutor，可把「订单执行策略」嵌进「组合管理策略」联合优化；
  • 核心代码：qlib/rl/（simulator、trainer、reward、order_execution…）。

15. 在线服务与模型滚动

研究模型最终要面对「每天新数据到来」的生产场景。Qlib 提供：

  • Online Serving：在线预测更新；
  • Automatic Model Rolling：按时间窗口滚动重训 / 更新；
  • 示例：examples/online_srv/（update_online_pred.py、rolling_online_management.py 等）。

相关模块：qlib/workflow/online/、qlib/contrib/online/、qlib/contrib/rolling/。

18. 市场动态适应

金融市场非平稳：训练集上学到的模式可能在未来失效。Qlib 提供：

  • Rolling Retraining：滚动窗口重训基线（examples/benchmarks_dynamic/baseline/）
  • DDG-DA：AAAI 2022，基于元学习/分布自适应的方法 （examples/benchmarks_dynamic/DDG-DA/）

元学习相关代码：qlib/model/meta/、qlib/contrib/meta/。

19. RD-Agent 联动（最新方向）

RD-Agent 是微软开源的 LLM 驱动自主研发 Agent，可与 Qlib 结合做 自动化因子挖掘与模型优化，朝「自动量化工厂」演进。
  • 仓库：microsoft/RD-Agent
  • 论文：R&D-Agent-Quant（arXiv:2505.15155）
  • 能力：从研报挖因子、量化因子搜索、模型超参/结构优化等

若你的目标是「让 AI 帮写因子 / 调模型」，应同时关注 Qlib（执行与回测底座）+ RD-Agent（研发 Agent 层）。

20. 注意与 FAQ

20.1 常见坑

包名 pip 安装用 pyqlib，代码里 import qlib。不要 pip install qlib（可能装到错误包）。
运行目录 执行 qrun 时建议 cd examples，避免工作目录下本地 qlib 包导入冲突。
未来函数 使用财报、一致预期等低频基本面时务必考虑 PIT；回测成交价、涨跌停、停牌假设要符合目标市场。
pandas 兼容 pandas 1.5→2.0 的 groupby(group_key=...) 默认值变化曾影响部分脚本； 个别示例（如部分 RL / TRA / TFT）需关注版本约束。
模型环境 不同 benchmark 依赖不同（如旧 TFT 依赖 TF 1.15）。批量脚本会为模型建独立虚拟环境（Linux）。

20.2 离线 / 在线数据模式怎么选？

  • 个人笔记本研究 → Offline + 本地 ~/.qlib/qlib_data/
  • 多人共享同一份干净数据 → Online Data Server

20.3 如何换成自己的股票池 / 市场？

  1. 把行情转为 Qlib .bin（dump_bin.py）；
  2. qlib.init(provider_uri=..., region=...)；
  3. 在 Handler 配置里改 instruments、时间段、benchmark；
  4. 调整回测费率与涨跌停规则以匹配市场（港股/美股与 A 股不同）。

20.4 与「本地持仓系统」的边界

Qlib 擅长研究与回测；真实成交、券商接口、本地账本通常由你自己的系统 （例如个人持仓库）负责。常见集成方式：用 Qlib 产出信号/权重，再由交易执行层下单， 持仓与成交回写到自有数据库做复盘。
