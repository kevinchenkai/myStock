富途 API 数据扩展调研与升级计划

评估 futu OpenAPI 还有哪些数据可获取，并经本地 OpenD 实测验证可行性 · 2026-07-18

调研目的与方法

目标：在不破坏现有架构边界（采集层抓取 → SQLite → Web 只读）的前提下，找出富途 OpenAPI 中当前尚未接入、且能实质提升产品价值的数据源，给出可执行的优先级建议。

方法：

  1. 通读现有 mystock/collectors/futu_client.py 与 schema.sql，确认已接入的富途接口与数据表；
  2. 调研富途官方文档（行情接口 / 交易接口总览及各子接口页），逐一比对字段、市场支持（仅关心 HK / US）、限频、以及是否需要付费行情权限；
  3. 按「产品价值 × 落地成本 × 边界契合度」三维打分，产出优先级。
评估口径 本项目是个人持仓复盘工具，非实时交易终端。因此评估天然偏向：账户/持仓维度的增量信息 > 实时盘口类高频数据。凡需要付费实时行情权限、或只有当日实时快照而无历史沉淀能力的接口，价值大打折扣。

A · 账户资金 accinfo_query P0 · 实测通过

能拿到什么

实时账户快照，含资产、现金、购买力，且按币种拆分（正好契合本项目 HKD/USD 不混算的口径）：

字段含义
total_assets账户净资产（证券 + 现金 + 债券）
market_val持仓证券市值
hk_cash / us_cash港币 / 美元现金（按币种）
frozen_cash冻结资金
avl_withdrawal_cash可提现金
power、hkd_net_cash_power / usd_net_cash_power最大购买力 / 现金购买力（按币种）
risk_status风险等级（LEVEL1–9）

可行性 （实测✅）

  • 属交易接口，复用现有 OpenSecTradeContext；实测无需交易密码即可读（未 unlock 也返回数据）。
  • 限频宽松（刷新缓存时每账户 30 秒 10 次；本项目每天 1 次，毫无压力）。
  • 与现有 query_positions 几乎同源，改动集中在采集层，成本很低。
  • 实测：总资产 / 持仓市值 / 现金 / 购买力 / 风险等级（LEVEL3）等字段均正常返回。（金额涉隐私，此处略）

产品价值 ★★★★★

  • 把"持仓市值"升级为"账户净资产"：组合概览卡新增「总资产 / 现金 / 持仓占比 / 购买力」，与券商 App 对齐。
  • 资产趋势更真实：新增账户净资产曲线，能反映现金进出（现有快照做不到）。
  • 现金比例：可算"仓位"（持仓市值 / 总资产），复盘"满仓/空仓时点"。
落地建议（据实测账户模型） 因只有一个 HK+US 综合账户，新增 account_funds 表按每日一条快照存： 
account_funds(
  snapshot_date TEXT PRIMARY KEY,   -- 每天一条
  report_currency TEXT,             -- 记账币种(HKD)
  total_assets REAL,                -- 合并总资产
  market_val REAL, cash REAL, frozen_cash REAL,
  power REAL,                       -- 最大购买力
  hkd_assets REAL, hk_cash REAL,    -- 港币侧
  usd_assets REAL, us_cash REAL,    -- 美元侧
  risk_status TEXT, updated_at TEXT
)
随 update.sh 每天抓一份（限频 30s/10 次，毫无压力）。前端组合概览新增「总资产 / 现金 / 仓位 / 购买力」，资产趋势加"账户净资产"线。投入产出比最高。

B · 资金流向 get_capital_flow P1 · 实测通过

能拿到什么

个股资金净流入，按订单大小分层，且提供历史日频（近 1 年）——这点很关键，能沉淀入库：

字段含义
in_flow整体净流入
main_in_flow主力净流入（仅历史周期）
super_in_flow / big_in_flow超大单 / 大单净流入
mid_in_flow / sml_in_flow中单 / 小单净流入
capital_flow_item_time时间（日频时为日期）

可行性 （实测✅ HK+US 均通过）

  • 行情接口，PeriodType.DAY 实测：HK.00700 返回 244 天、US.AAPL 251 天日频历史。
  • 限频 30 秒 30 次（本项目 38 只标的、每天 1 次，绰绰有余）；美股也支持，无需担心权限。
  • 中等成本：新增采集函数 + capital_flow 表（主键 code+date）+ 个股详情页一个展示位。

产品价值 ★★★★☆

  • 个股详情页新增「主力资金流向」柱状/折线：直观看"聪明钱"进出，是持仓复盘的强补充。
  • 可与 K 线叠加，观察"放量上涨是否有主力承接"。
  • 近 1 年历史可一次性回补入库，之后随 update 增量——完全符合"沉淀式"数据模式。

C · 市场快照 get_market_snapshot P2 可选

能拿到什么

实时快照，字段远比 yfinance info 丰富且更"盘面化"：turnover_rate（换手率）、amplitude（振幅）、volume_ratio（量比）、bid_ask_ratio（委比）、highest52weeks_price / lowest52weeks_price（52 周高低）、pe_ratio / pb_ratio / pe_ttm_ratio、dividend_ttm、total_market_val、lot_size 等。批量最多 400 标的/次，限频 30 秒 60 次。

价值与取舍 ★★★☆☆

  • 换手率 / 振幅 / 量比 / 52 周高低是 yfinance 缺的、对复盘有用的盘面指标，可补进 stock_profiles 或新表。
  • 但是：它是实时快照，无历史——只能每天抓一份沉淀，且估值字段与现有 yfinance stock_profiles 高度重叠。
  • 结论：非刚需。若已做 A/B，可顺带把"换手率/振幅/52 周高低"三个 yfinance 没有的字段补进来，增量小。

D · 所属板块 get_owner_plate P2 可选

返回个股所属的行业 / 概念板块（plate_code / plate_name / plate_type）。限频 30 秒 10 次，每次 ≤200 标的。

价值 ★★★☆☆

  • 可给持仓做行业/概念聚类：组合概览新增"行业分布"（如科技/消费/新能源占比），比现有"美股/港股"更有洞察。
  • yfinance 的 sector/industry 已提供粗粒度行业，富途板块胜在概念板块（如"AI 概念""港股通"）与本地化命名。
  • 实测：HK 与 US 均返回丰富板块（文档说"美股未明确"过于保守）。腾讯命中"云计算/AI应用/元宇宙"，苹果命中"消费电子(行业)/巴菲特持仓/标普500十大"。

E · 历史 K 线 request_history_kline 不推荐替换 yfinance

关键限制：付费配额 官方明确：历史 K 线配额按账户资产与交易活跃度分配，「7 天内只能拉取有限只股票的历史 K 线」，消耗的配额 7 天后才释放。这不是免费的 yfinance 替代品——无法批量下载 38 只标的的历史日线。
  • 字段确实更全（含 turnover_rate / change_rate / pe_ratio），数据深度也好（日 K 近 20 年）。
  • 但配额模型决定了它不适合本项目"全持仓每日刷新"的用法。
  • 结论：维持现状用 yfinance 抓日线（已加限频节流防护）。富途历史 K 线仅在"某只重点股要看更长历史/更全字段"时按需点抓，不做常规管线。

风险与边界

  • 权限已实测排除：资金流向 / 快照 / 板块在当前账户 HK+US 均返回数据，账户资金无需交易密码即可读——原先"最大不确定性"已消除。但若日后更换账户/降级行情权限，仍需重跑探针。
  • OpenD 依赖：这些都是富途接口，抓取时 OpenD 必须已启动并登录（与现有 update.sh 前置条件一致）；OpenD 未开时优雅跳过、记 sync_log，不中断 yfinance 部分。
  • 坚守架构边界：所有新数据一律走"采集层抓 → 入 SQLite → Web 只读"。前端绝不直连富途。新表沿用 UPSERT 幂等 + 每日快照/增量的既有模式。
  • 限频复用现有机制：交易接口沿用 _query_with_retry 的退避；行情接口新增时也要加对应节流（参照 yfinance 那次限频教训）。
  • 历史不可回补的仍不可回补：账户资金、快照都是"当日快照"，历史空缺只能随 update.sh 自然积累——与 positions 同理，需在文档中写明。
  • 隐私：账户资金是更敏感的数据，仍只入 data/mystock.db（已 gitignore），绝不进 git。

一句话总结：4 个候选接口已本地 OpenD 实测全部可行（HK+US、无需交易密码）。第一期做 accinfo_query（账户净资产，价值最高）；第二期做资金流向（已验证近 1 年日频可入库）；第三期打包板块聚类 + 盘面增量字段。与 yfinance 重叠的（历史日线、PE/PB/市值/股息）不重复抓，各取所长。
